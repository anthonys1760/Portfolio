Tech decisions

Short architectural decision records — the tradeoffs behind the shipped work.

React Native + Expo over native Swift for NutriLogix 2025 · NutriLogix AI

Context. TODO: Solo founder, need iOS + Android + web from one codebase, fastest path to TestFlight beta.

Decision. TODO: Expo-managed React Native with EAS builds, OTA updates via expo-updates.

Tradeoffs. TODO: Slightly heavier bundle and some native-module friction vs. 3–4x faster ship velocity and a single TypeScript codebase across platforms.

Supabase over Firebase for auth + storage 2025 · NutriLogix · VoiceLift

Context. TODO: Need auth, Postgres, row-level security, and file storage without standing up a backend team.

Decision. TODO: Supabase — Postgres with RLS policies, edge functions for server-side logic, storage buckets signed per user.

Tradeoffs. TODO: Lose Firebase's real-time sync maturity and offline-first SDK; gain SQL, relational modeling, and an escape hatch to self-hosted Postgres later.

Provider-abstraction layer for AstroAI X 2025 · AstroAI X

Context. TODO: Users want to switch between GPT-5, Claude Opus 4, and Gemini 2.5 Pro mid-conversation without losing context.

Decision. TODO: Single chat-message interface + per-provider adapter that normalizes streaming, tool calls, and token accounting. Context persisted in a shared conversation store, replayed per provider.

Tradeoffs. TODO: More adapter surface to maintain vs. locking the product into one provider's roadmap and pricing.

Whisper + parser pipeline over structured voice forms 2025 · VoiceLift

Context. TODO: Mid-workout users can't tap through forms. Need to capture sets, reps, supersets, and gym slang hands-free.

Decision. TODO: Whisper for transcription → small LLM parser tuned on gym vocabulary → deterministic set/superset state machine → Supabase log. Local fallback when offline.

Tradeoffs. TODO: Higher per-log cost than a structured form, but friction goes to near-zero — which is the entire product thesis.

GTM + GA only — no session replay or chat widgets 2026 · anthonysdigital.net

Context. TODO: Inherited template had Hotjar, Mouseflow, Google Optimize, Chatbase, ManyChat, and orphan debug scripts. Privacy + performance cost was high, signal was low.

Decision. TODO: Strip every third-party script except GTM + GA. Replace Chatbase with a vanilla-JS streamed chat hero I own end-to-end.

Tradeoffs. TODO: Lose heatmaps and replay, gain trust, speed, and zero vendor lock-in on the hero interaction.
